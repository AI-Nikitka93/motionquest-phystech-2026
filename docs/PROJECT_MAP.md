# Project Map

## Goal
Build a PhysTech 2026 hackathon submission that can compete on Impact, Creativity and Presentation.

## Current Product Direction
MotionQuest: an adaptive home movement lab for older adults and seated/standing users, using research-aligned functional movement tasks and caregiver-readable reports.

## Core Scope
- 2 activities: Adaptive Chair Movement and Reach Stars.
- Adaptive Chair Movement has two branches: standing sit-to-stand and seated-adaptive upper-body movement.
- 1 Caregiver Report screen.
- Client-only web app.

## Implementation Structure
- `motionquest-app/src/app/` - Next.js App Router entrypoint.
- `motionquest-app/src/components/` - MotionQuest screens and camera stage.
- `motionquest-app/src/hooks/usePoseTracking.ts` - webcam, MediaPipe Pose and MediaPipe HandLandmarker integration.
- `motionquest-app/src/lib/gameLogic.ts` - pure Adaptive Chair Movement / Reach Stars heuristics.
- `motionquest-app/src/lib/sessionStorage.ts` - localStorage session persistence.
- `motionquest-app/src/lib/visualSystem.ts` - visible trust, evidence, icon, asset and provenance content used by the app.
- `motionquest-app/tailwind.config.ts` - visual tokens from `docs/visual-spec.md`.

## Build Enablement And Agent Workflow
- `AGENTS.md` is the first project instruction surface for future agents.
- `motionquest-app/scripts/project-readiness.mjs` verifies required project/app files, required npm scripts, the no-backend/no-auth/no-database MVP boundary and obvious token patterns across instructions/docs/src.
- `motionquest-app/scripts/final-submission-audit.mjs` separates `local_package`, `public_publication`, `external_submission`, `real_camera_evidence` and `final_submission` so local readiness cannot be mistaken for actual submission readiness.
- `motionquest-app/package.json` exposes the check as `npm run project:readiness`.
- `motionquest-app/package.json` exposes the final-submission boundary check as `npm run project:final-audit`; add `-- --public-smoke` when public HTTP status should be refreshed.
- Skill discovery decision: `npx --yes autoskills --dry-run` is allowed for manual review, but no third-party skill is installed unless it directly helps a current MotionQuest task and the install decision is recorded in `docs/DECISIONS.md`.
- Current codebase-intelligence decision: no external indexer is required for this client-only app; use `rg`, exact-string search for known identifiers, existing tests and targeted full-file reads.

## Risks
- Browser pose tracking calibration may be fragile.
- Pose overlay now rejects single-limb/partial-body noise and supports seated/reach tracking without requiring hips or shoulders for seated-adaptive/reach branches. Seated/reach modes ignore PoseLandmarker body/arm landmarks and accept only real HandLandmarker wrist data. Standing branch now pauses timer/scoring until usable full-body tracking exists. Real-room camera distance, lighting and framing still need physical smoke testing.
- Webcam permission and lighting can break live demo.
- Clinical claims must stay conservative.
- Dev server uses port `3001` in this workspace because `3000` was already occupied during verification.
- Public source/storage package and demo media are created; real webcam evidence and final Devpost submission remain future blockers.
- T086-T089 cannot be fully closed in this environment until a real physical webcam or real camera evidence is available; seated-adaptive mode is implemented and deployed but still needs user-side live-camera tuning evidence.
